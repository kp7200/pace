# 📏 UI Agent Rules

## 🎯 Purpose
Define strict behavioral constraints for UI generation to ensure consistency, clarity, and usability.

---

## 🧱 Core Rules

### 1. Clarity Over Creativity
- Always prioritize usability over visual experimentation
- Avoid decorative elements that do not add value
- Ensure every element has a clear purpose

---

### 2. Single Primary Focus
- Each screen must have one clear primary focus
- The most important information must dominate visually
- Secondary elements must never compete with primary content

---

### 3. Minimalism Enforcement
- Remove any non-essential UI elements
- Avoid feature stacking in a single screen
- Prefer fewer elements with clearer meaning

---

### 4. No Cognitive Overload
- Limit the amount of visible information
- Avoid requiring mental calculations by the user
- Present computed values directly

---

### 5. Consistent Layout Behavior
- Maintain consistent spacing, alignment, and structure
- Avoid unpredictable layout variations
- Keep interaction patterns uniform

---

### 6. Immediate Understandability
- User must understand the screen within 2 seconds
- No hidden meaning or unclear indicators
- Avoid ambiguous labels or icons

---

### 7. Action Visibility
- Primary actions must be clearly visible
- Avoid hiding critical actions behind menus
- Ensure actions are easy to reach and interact with

---

### 8. State Transparency
- Always reflect current state clearly:
  - Active / inactive
  - Completed / pending
- Avoid silent state changes

---

### 9. Input Simplicity
- Minimize effort required for user input
- Avoid multi-step interactions when unnecessary
- Prefer inline or quick-entry methods

---

### 10. Theme Consistency
- Maintain identical layout structure across themes
- Ensure readability in all modes
- Avoid visual inconsistencies between themes

---

## ⚠️ Strict Prohibitions

- ❌ Do not introduce charts or analytics unless explicitly required
- ❌ Do not add multiple navigation layers for simple workflows
- ❌ Do not overload UI with statistics
- ❌ Do not use complex gestures as primary interactions
- ❌ Do not design for edge cases before core use is complete

---

## 🔁 Refinement Rules

- Always simplify after initial generation
- Reduce UI elements wherever possible
- Strengthen hierarchy in each iteration
- Eliminate redundancy

---

## 🧪 Validation Checklist

Before finalizing any UI:

- Can the user understand it instantly?
- Is the primary information dominant?
- Is any element unnecessary?
- Are actions obvious?
- Is the layout clean and breathable?

If any answer is “no”, refine further.

---

## 🏁 Outcome

The UI Agent must produce interfaces that are:

- Predictable  
- Minimal  
- Clear  
- Efficient  

and optimized for **daily repeated usage without friction**.
